# Editing the Knowledge Base

This guide is for **editors** — no prior Astro or command-line knowledge needed.

All pages are plain Markdown files under `src/content/docs/`. The folder
structure maps directly to the URL: `src/content/docs/help/first-steps.md`
becomes `/help/first-steps/`.

## Option A — quick edits in the browser (GitHub)

Best for fixing a typo, updating a sentence, or adding a paragraph.

1. Open the page file on GitHub under `src/content/docs/…`.
2. Click the **pencil icon** (Edit this file).
3. Make your change in Markdown.
4. Scroll down, write a short description, and click **Commit changes**.
5. The site rebuilds and publishes automatically (a few minutes).

## Option B — full local preview

Best for larger edits, adding images, or checking how it looks before publishing.

```bash
nvm use 22
npm install       # first time only
npm run dev       # opens http://localhost:4321/cnr-kb-poc
```

Edit files under `src/content/docs/`; the browser reloads instantly. When happy,
commit and push (or open a pull request).

## Page frontmatter

Every page starts with a frontmatter block between `---` lines:

```yaml
---
title: "Payment Settings"      # page title + sidebar entry
description: "How to configure payment methods."  # optional, for SEO
sidebar:
  order: 40                    # position in the sidebar (lower = higher up)
  label: "Payments"            # optional shorter sidebar label
---
```

- **Reorder the sidebar:** change `sidebar.order`. Items are sorted by this
  number within their group; leave it out to sort alphabetically at the end.
- **Rename a sidebar entry** without changing the page title: set `sidebar.label`.

## Writing content

Standard Markdown works (headings, lists, links, tables, images). A few
KB-specific building blocks are available via **directives** — you rarely need
to write these by hand (the migration generates them), but you can:

````markdown
:::command[Command]
```text
command = AddDomain
domain  = yourdomain.com
```
:::

:::exception[Exception]
```text
command = AddDomain
domain  = yourdomain.de
X-DE-ACCEPT-TRUSTEE-TAC = 1|2
```
:::

:::gateways
- [MREG](../connecting/centralnic-reseller-metaregistry/)
- [HTTPS](../connecting/hypertext-transfer-protocol-socket/)
:::
````

- `:::command` / `:::response` — yellow / blue API boxes (with copy button).
- `:::exception` — red box for exceptions.
- `:::gateways` — the coloured gateway pill badges.
- `:::commandlist[Category]` — a row of neutral command buttons.

Starlight's own components ([asides](https://starlight.astro.build/guides/authoring-content/#asides),
cards, etc.) also work — e.g. `:::note`, `:::tip`, `:::caution`.

## Images

Put images in `public/media/…` and reference them from a page as
`/cnr-kb-poc/media/your-image.png`. They are served from the site itself (no
external dependency).

## Adding a new page

1. Create a `.md` file in the right folder under `src/content/docs/`.
2. Add a frontmatter block with at least a `title`.
3. Write your content. It appears in the sidebar automatically.

## TLD pages

TLD pages under `domains/tlds/` are **generated** from the registry source by
`scripts/migrate.mjs` — don't hand-edit them; changes would be overwritten on
the next migration. The homepage and TLD-landing search box is driven by
`src/tlds.json`, regenerated by `scripts/gen-tlds.mjs`.
